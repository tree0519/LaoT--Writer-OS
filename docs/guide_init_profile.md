# 初始化配置向导详解 (/init-profile)

`/init-profile` 是 老T-Writer-OS 的核心启动命令。它就像游戏的“捏脸”系统，帮助你创建一个独一无二的 AI 写作助手。

## 1. 为什么需要它？

普通的 AI 写作往往千篇一律，因为它们不知道“你是谁”。
`/init-profile` 的作用是固化你的：
*   **笔名**：让文章有署名感。
*   **人设**：决定了看待问题的角度（例如：程序员看世界 vs 产品经理看世界）。
*   **风格**：决定了文字的调性（犀利、幽默、温柔等）。
*   **背景**：增加可信度（例如：10年大厂经验）。

## 2. 使用流程详解

在终端输入 `/init-profile` 后，系统会启动交互式问答。

### 第一步：输入笔名
> `1. 请输入您的笔名/昵称 (如: 老T):`
*   **作用**：用于文章署名或自称。
*   **建议**：使用你常用的社交媒体昵称。

### 第二步：定义人设
> `2. 请输入您的核心身份/人设 (如: 资深全栈工程师, 毒舌产品经理):`
*   **作用**：AI 会扮演这个角色进行思考。
*   **示例**：
    *   `资深全栈工程师` -> 关注技术细节，逻辑严密。
    *   `创业失败的连续创业者` -> 充满实战经验，可能有悲情色彩。
    *   `情感博主` -> 细腻，感性，关注人际关系。

### 第三步：设定风格
> `3. 请描述您的写作风格 (如: 犀利、幽默、拒绝说教):`
*   **作用**：调整 AI 的语气（Tone）。
*   **关键词推荐**：
    *   `犀利`：观点鲜明，敢于抨击。
    *   `幽默`：多用梗，轻松活泼。
    *   `干货`：由数据和案例堆砌，少废话。
    *   `治愈`：温暖，正能量。
    *   `拒绝说教`：避免“综上所述”、“小编觉得”等 AI 味。

### 第四步：背景故事
> `4. 一句话描述您的背景 (如: 10年大厂被裁经验):`
*   **作用**：为文章提供“真实感”素材。AI 可能会在文章中随机引用这些背景。
*   **示例**：`曾任职于字节跳动，负责过千万级并发系统设计。`

### 第五步：选择平台
> `5. 请选择要启用的平台 (用逗号分隔，回车默认全部):`
*   系统会列出当前可用的平台模板（例如：`xiaohongshu`, `zhihu`, `wechat_public` 等）。
*   **全部启用**：直接按回车。
*   **部分启用**：输入 `xiaohongshu, zhihu`。
*   **自定义平台**：如果你输入了一个不存在的平台名（例如 `bilibili`），系统会自动为你创建一个基于“万能模板”的配置。

## 3. 进阶玩法：添加新平台

如果你想让 AI 学会写 B 站文案或推特推文，不需要修改代码！

1.  **极简方式**：
    *   在 `templates/` 目录下新建一个文件 `bilibili_skeleton.md`。
    *   在文件里写上你对 B 站文案的要求（记得包含 `{{user_role}}` 占位符）。
    *   再次运行 `/init-profile`，你会发现 `bilibili` 已经出现在可选列表中了！

2.  **推荐方式**（更友好）：
    *   编辑 `config/platform_registry.json`，添加：
        ```json
        "bilibili": {
            "name": "哔哩哔哩",
            "description": "年轻化，玩梗，弹幕互动",
            "template_file": "bilibili_skeleton.md"
        }
        ```
    *   创建对应的 `templates/bilibili_skeleton.md`。
    *   运行 `/init-profile`，向导会显示中文名称“哔哩哔哩”。

## 4. 常见问题

*   **Q: 我填错信息了怎么办？**
    *   A: 随时重新运行 `/init-profile` 覆盖即可。或者直接修改 `config/source_persona.json` 文件。
*   **Q: 生成的 AI 还是不够像我怎么办？**
    *   A: 尝试在“写作风格”里加入更具体的形容词，或者提供一段你的代表作片段。
